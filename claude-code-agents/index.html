Setup — two ways in

pick the path that matches what you have open

Terminal path

Claude Code CLI in VS Code / Terminal / PowerShell.

  1. Create a folder on your Desktop named lesson-sandbox (right-click → New Folder).
  2. Mac: right-click the folder → "New Terminal at Folder." · Windows: right-click inside it in Explorer → "Open in Terminal."
  3. Type claude + Enter.

Desktop App path

Claude Code Desktop App on Mac or Windows.

  1. Create a folder on your Desktop named lesson-sandbox.
  2. Open the Desktop App → click the Code tab.
  3. Select Local → Select folder → pick lesson-sandbox.
  4. Pick a model (Sonnet is fine).
Approving file writes: Terminal → press y. Desktop → click Accept on the diff.
Seeing your files: Terminal → use Finder/Explorer. Desktop → click the file icon in the sidebar.
01

What is an LLM?

a prediction machine · the engine inside every agent

An LLM is a prediction machine. Trained on enormous datasets, it does one thing: given some input, predict the most likely next output. That's the entire engine. What changes from chatbot to agent is the range of things we let it predict.

"The sky is…"
→
LLM
→
blue89%
cloudy5%
falling3%
The "harness" — what we let the LLM predict
1 Chat
You type → LLM → predicts text backThe original interaction layer. ChatGPT in 2022.
Input is text. The only output it can predict is more text.
↓ expand the prediction space ↓
2 Chat + tools
You type → LLM → predicts text OR a tool callread a file · search the web · call an API · run code
Same LLM. We just gave it more options for what to predict next.
↓ expand further ↓
3 Agent
You give a goal → LLM → predicts the next step, in a looptext · tool · observe result · ↻ adjust · ↻ another tool · text · done
Same prediction engine, just looping with a much bigger output space. That's an agent.
The LLM doesn't change. What changes is the harness around it — the menu of choices we let it predict. The next section is about that third row.

What even is an "agent"?

working definitions · the parts · examples in the wild

The word "agent" is overloaded. Three definitions that overlap and capture the 2026 consensus:

Plain
"An LLM using tools, in a loop."
Anthropic
"A system where the model dynamically directs its own process and tool use." Vs. a workflow, which follows a predefined sequence.
Practical
"Software that takes a high-level goal, breaks it into steps, and gets the work done without step-by-step supervision."

Workflow & business embedded in your tools

05

CLAUDE.md

project memory · loaded every session

A plain markdown file Claude reads when you start a session. It's just a text file sitting in a folder. There are two places Claude looks — your home folder, and the project folder you opened Claude Code in:

Your home folder
~/
└── .claude/
└── CLAUDE.mdpersonal · all projects
+ both get loaded when you open a project ↓
A project folder
~/Desktop/lesson-sandbox/
├── CLAUDE.mdproject · team-shared
└── CLAUDE.local.mdlocal · gitignored
Both files combine. If they conflict, the project file wins. Repeat the same correction every session? It belongs here.

06

Commands

your shortcuts · you trigger them

A markdown file at .claude/commands/<name>.md. You type /name and the file's contents become the prompt. $ARGUMENTS stands in for whatever you typed after.

You type/greet matt
→
File body becomes promptGreet matt in pirate-speak…
Commands bottle up "the prompt I keep retyping." Checked into the folder — your teammates get them for free.

Try it · build /greet

.claude/commands/greet.md
--- description: Greet someone in pirate-speak --- Greet $ARGUMENTS in the most piratical voice you can muster. Use no more than three sentences.
1
Tell Claude: "Create .claude/commands/greet.md with the content above."
2
Approve the file write.
3
Type /greet your-name
Share: drop your pirate greeting in chat.
07

Skills

the modern unified primitive · Claude triggers them

A directory at .claude/skills/<name>/SKILL.md. The description field is the matchmaker — Claude scans descriptions to decide which skill to run when you ask something. You never name the skill.

"make this rhyme: I love coffee in the morning"
tldrSummarizes long text into bullets.
rhyme-itRewrites a sentence to rhyme.
translateTranslates between languages.
Claude scans descriptions → finds the match → runs that skill.
Three frontmatter flavors: default (both invoke) · disable-model-invocation: true (you only — what "commands" used to be) · user-invocable: false (Claude only). Follows the Agent Skills open standard.

Try it · build rhyme-it

.claude/skills/rhyme-it/SKILL.md
--- description: Rewrites a sentence to rhyme. Use when the user asks to make something rhyme. --- Rewrite so it rhymes. Keep the meaning. Reshape the last words. Return only the rewritten version.
1
Tell Claude: "Create .claude/skills/rhyme-it/SKILL.md with the content above."
2
Approve.
3
Don't name the skill. Just ask: "make this rhyme: I love coffee in the morning."
Share: post the rhymed version.

Subagents

fresh context · focused job · dispatched

A file at .claude/agents/<name>.md. The main Claude dispatches the work to a fresh context window; the subagent does the job and returns only the result. Your main thread stays clean.

Main conversation
"Use the explainer agent to explain APIs."
→
Subagent · fresh context
Explainer works in isolation, returns a 200-word answer.
↑ only the answer comes back to your main thread ↑
Built-in types: Explore, Plan, general-purpose, Bash. Custom ones live in .claude/agents/.

Try it · build explainer

.claude/agents/explainer.md
--- name: explainer description: Explains a topic in beginner-friendly language. --- You are a patient teacher. Given a topic, explain in ≤200 words using plain language + one good analogy. Define any jargon.
1
Tell Claude: "Create .claude/agents/explainer.md with the content above."
2
Approve.
3

09

Subagents working together

a real workflow · main Claude orchestrating three subagents
A skill is…
A playbook Claude loads into its own context when something matches. Guidance Claude follows while staying in your conversation.
vs
A subagent is…
A worker Claude dispatches to a separate fresh context. It does the job alone and returns only the result. Defined by an agent.md file.

One subagent is useful. Several subagents working in concert is where decomposition pays off. You give a goal; main Claude breaks it into pieces and dispatches each piece to a fresh-context worker. Your main conversation only ever sees the summaries.

YOU
"Write me a blog post on prompt engineering for beginners."
↓goal
MAIN CLAUDE
Decomposes: research → draft → critique. Dispatches three subagents via the Task tool, each with their own brief.
↓ ↓ ↓dispatch via Task tool
Researcher
.claude/agents/researcher.md
Reads 10+ sources on prompt engineering, ranks the best 5, summarizes each.
↳ Returns: 5-bullet summary. The 10+ raw articles stay in its own context.
Writer
.claude/agents/writer.md
Drafts a 600-word post in a friendly, beginner-aimed voice from the researcher's summary.
↳ Returns: a clean draft. Drafting iterations stay in its own context.
Critic
.claude/agents/critic.md
Reviews the draft for accuracy, jargon, and beginner-friendliness.
↳ Returns: revision notes. The line-by-line scan stays in its own context.
↑ ↑ ↑only summaries return
MAIN CLAUDE
Has: a 5-bullet research summary, a draft, and revision notes. Applies the notes to the draft. Produces the final post.
↓polished post
YOU
Receive only the finished post. The 10+ sources, the rough draft, the back-and-forth — none of it cluttered your thread.
What's happening: three agent.md files in .claude/agents/ defined three workers. Main Claude (acting on your behalf) decomposed the goal and dispatched each worker via the Task tool. Each ran in its own fresh context, did its job, returned a summary. Your main thread stayed clean. This is the pattern that makes complex workflows tractable.

★

Recap — which one do I reach for?

a decision tree you'll actually use

You've built one of each pillar. When you're back at your own work and asking "which file belongs here?" — walk this tree:

Is it always relevant in this project?
→
CLAUDE.md
Do I want to invoke it by typing a slash?
→
.claude/commands/<name>.md
Do I want Claude to invoke it when something matches?
→
.claude/skills/<name>/SKILL.md
Does the work need a fresh context window?
→
.claude/agents/<name>.md
The shape mirrors the file tree from §4. Anything in CLAUDE.md is always there. Anything in .claude/ only runs when something triggers it — you, Claude, or a dispatch.
